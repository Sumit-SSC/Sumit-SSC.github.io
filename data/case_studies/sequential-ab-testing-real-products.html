Fixed-horizon A/B tests force a choice: wait for a pre-set sample size (slow) or peek and stop early (biased). Sequential A/B testing allows early stopping with valid inference by using pre-defined boundaries. This case study covers sequential testing in real products, early-stopping bias, and guardrails—using Netflix and arXiv.

Executive Summary

  • Sequential testing: Compute test statistic at interim looks; compare to upper (reject H0) and lower (accept H0) boundaries; stop when a boundary is crossed, else continue.
  • Early stopping bias: Naive peeking inflates Type I error; sequential boundaries are designed to preserve Type I/II error while allowing early stop when effect is clear.
  • Guardrails: Pre-specify primary metric and boundaries; monitor guardrail metrics at each look; do not use sequential design as excuse to peek on many metrics.
  • Implementation: Use validated sequential methods (e.g., O'Brien-Fleming, Pocock, or custom boundaries); document analysis plan upfront.

Business Context & Problem Framing

Product teams want fast answers; fixed-horizon tests can feel slow. Netflix's experimentation posts describe sequential testing as a way to conclude experiments 30–50% faster on average while controlling false positives. The key is using proper boundaries (e.g., from group-sequential design) rather than ad-hoc peeking.

Data Landscape & Event Model

Standard experiment schema: assignment (user_id, experiment_id, arm, assigned_at), events (user_id, event_type, value, timestamp). For sequential analysis, aggregate to experiment-level metric (e.g., mean per arm) at each look; compute test statistic (e.g., Z or likelihood ratio) and compare to boundaries. Looks are typically time-based (e.g., daily) or sample-size-based.

Metrics, KPIs & North Star

Primary: One metric for the sequential test (e.g., conversion rate, retention). Guardrails: Metrics checked at each look; if a guardrail breaches, consider stopping for harm even if primary has not crossed a boundary. North star: correct ship/no-ship decision with minimum expected sample size.

Analytical / Experimental Approach

Pre-specify: primary metric, null and alternative, Type I/II error, maximum sample size, and boundary type (e.g., O'Brien-Fleming). At each look: compute point estimate and variance per arm, then test statistic; compare to boundaries; stop if crossed, else continue. Report final decision and confidence interval (e.g., repeated-confidence-interval for sequential).

AI & Automation Layer

Automated boundary computation and daily (or per-look) evaluation; dashboards showing statistic path and boundaries; alerts when boundary crossed or guardrail breached; optional LLM summarization of "experiment concluded: ship treatment, primary +X%, guardrails OK."

Observations & Key Findings

Sequential testing reduces expected sample size for a given power when effects are large; early-stopping bias is controlled by design. Guardrails must be specified in advance; peeking on many metrics still inflates Type I error unless adjusted (e.g., Bonferroni on number of metrics).

Trade-offs, Risks & Failure Modes

Trade-off: speed vs. complexity (boundaries, analysis plan). Risk: misuse (e.g., changing primary metric mid-test). Failure mode: treating sequential as "peek until significant" without proper boundaries.

Business Impact & Decision Outcomes

Decision: adopt sequential testing for high-volume product experiments; pre-register primary and guardrails; use validated boundaries. Outcome: faster ship decisions with preserved statistical validity.

Limitations & What We'd Improve

Sequential methods assume correct model (e.g., normal); delayed outcomes complicate look timing. Improvements: sequential methods for ratio metrics and survival; integration with CUPED or variance reduction.

Future Scope (12–24 Months)

Platform-native sequential testing (e.g., Statsig, Eppo); multi-armed and bandit-sequential hybrids; pre-registration and audit trails.
